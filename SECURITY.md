# Security Policy

## Supported Versions

当前项目处于 `0.x` 开发阶段。安全修复优先进入主分支。

## Reporting a Vulnerability

请不要在公开 issue 中披露可利用细节。

建议通过以下方式报告：

- 私有安全报告功能，如果仓库平台支持
- 仓库维护者公开提供的安全邮箱

报告中请包含：

- 影响版本或 commit
- 复现步骤
- 影响范围
- 可行的修复建议，如果有

## Sensitive Data

不要在 issue、PR、截图或测试文件中提交：

- API Key
- Bearer Token
- 账户 ID
- 私有网页内容
- 生产环境日志
